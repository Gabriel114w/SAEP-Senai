# 9. Requisitos de Infraestrutura

A escolha da infraestrutura visa a robustez, escalabilidade e facilidade de manutenção, utilizando tecnologias de código aberto (open-source) e amplamente adotadas no mercado.

## Stack Tecnológica Proposta

| Categoria | Tecnologia Escolhida | Justificativa |
| :--- | :--- | :--- |
| **Sistema Operacional (OS)** | **Linux (Ubuntu Server)** | Estabilidade, segurança, baixo custo (gratuito) e vasto suporte da comunidade para ambientes de produção. |
| **SGBD** | **PostgreSQL** | SGBD relacional avançado, conhecido por sua robustez, conformidade com padrões SQL e capacidade de lidar com grandes volumes de dados e transações complexas. |
| **Linguagem de Programação** | **Python** | Linguagem de alto nível, legível e com vasta biblioteca, ideal para desenvolvimento rápido e manutenção de sistemas web. |
| **Framework (Backend)** | **Django** | Framework Python "batteries-included" que acelera o desenvolvimento, oferecendo ORM, sistema de autenticação (RF01) e painel administrativo prontos para uso. |
| **Framework (Frontend)** | **HTML5, CSS3, JavaScript (com Alpine.js ou similar)** | Uso de tecnologias web padrão. Um framework leve como Alpine.js ou HTMX pode ser usado para adicionar interatividade sem a complexidade de grandes frameworks como React ou Vue, mantendo a agilidade. |

## Recursos Mínimos de Hardware (Servidor de Aplicação e Banco de Dados)

Os requisitos mínimos são estimados para um ambiente de produção inicial com até 50 usuários simultâneos e um volume moderado de dados.

| Recurso | Mínimo Recomendado | Observação |
| :--- | :--- | :--- |
| **Processador (CPU)** | 2 vCPUs (x86-64) | Processadores modernos com boa performance single-core. |
| **Memória RAM** | 4 GB | Suficiente para o SGBD (PostgreSQL) e o servidor de aplicação (Django/Gunicorn/Nginx). |
| **Armazenamento (Disco)** | 100 GB (SSD) | SSD é crucial para a performance do banco de dados e do sistema operacional. 100GB para o SO, logs e dados iniciais. |
| **Conexão de Rede** | 100 Mbps | Conexão estável e dedicada para acesso dos usuários. |
